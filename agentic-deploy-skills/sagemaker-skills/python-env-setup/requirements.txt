# Core AWS dependencies for SageMaker deployment.
#
# No version pins. We want the latest of each — if scripts break against the
# latest, fix the script, not the version. Newer boto3/awscli/sagemaker-core
# have current API surfaces, more accurate URI resolution tables, and security
# fixes.
#
# Why sagemaker-core, not sagemaker:
#   - sagemaker-core is the v3 layer containing image_uris.retrieve() and
#     resource-shape definitions. We use it for image URI resolution (it
#     keeps current account IDs and tags per framework family).
#   - sagemaker (the meta-package) pulls in sagemaker-serve which contains
#     ModelBuilder — an opinionated builder that conflicts with our
#     explicit-stages design. We deliberately don't import from it.
#   - sagemaker-train is unrelated (training, not inference) and we don't
#     need it.
# See python-env-setup SKILL.md ("Why sagemaker-core only") for context.
#
# Add per-deployment extras (huggingface_hub for model inspection, transformers
# for tokenizer validation) to a deployment-specific requirements file alongside
# this one. Do not pin versions here unless you have a documented reason.

boto3
awscli
sagemaker-core
